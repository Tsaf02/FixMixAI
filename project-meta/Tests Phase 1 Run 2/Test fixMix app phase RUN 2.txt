Backend
The use of "Automatic Update" was tested, and here are the conclusions:

1. After opening FIXMIX via the terminal and activating the "eye icon" button, the terminal showed an operation that worked.
2. Attempting to write in the prompts' text box was completely disrupted. I couldn't write a single complete sentence before a disruption occurred: the text wouldn't write, the previously written text disappeared, and other disruptions likely originating from the operation via the UIA.
3. When I closed the "eye" icon, I was able to write a prompt, activate the "eye," send the prompt, and receive an answer from the AI in the original LLaVA - but the text did not update in the FIXMIX window.
4. I tried sending and receiving new text without turning off the eye, but the automatic update did not work without my intervention.
5. However, what did work was that when the "eye" icon was activated and a new text was finished being written in the chat, a click on "Copy" was enough, and immediately the text in FIXMIX was replaced with the new text. This new text only included the paragraph I copied. Because in AI-sourced chats in AI applications, there is a different "Copy" function for the text the user writes and a different one for the AI's responses, only the text I selected was pasted into FIXMIX - either the prompt I wrote or the answer. Never both together.

PS - The intention was that the previous text would not be deleted, but would be added to the bottom, continuing the chat, just like an AI chat behaves. In other words, the desire was for there to be a recognition of all text added since the last text that was copied, so that the user could scroll down and up and see a wide range of the chat or the artifact from right to left, as in the original display.
